# Requirements pip-installed into the AppImage's bundled Python, in order.
#
# IMPORTANT: python-appimage runs each `pip install <line>` through a shell
# (' '.join then shell=True), so a line must contain NO shell metacharacters —
# no spaces, no '>' / '<' (redirects), no '[...]' (globs), no '@'. That rules
# out PEP 508 "name @ url" forms, "pkg[extra]" extras, and ">=x.y" version
# pins. Bare names / a git+ URL / a plain path are safe; hence the forms below
# (and the bare checkout path appended by build.sh).
#
# photonfdtd (the FDTD solver) isn't on PyPI — install it from GitHub. First, so
# it's present before phidler needs it. A bare git URL (no ref) installs main's
# current HEAD, so every nightly bundles the newest solver; build.sh sets
# PIP_NO_CACHE_DIR (no stale wheel) and logs the resolved commit for provenance.
git+https://github.com/ngpaladi/photonfdtd.git
#
# The FDTD extras, listed explicitly (unpinned) instead of via phidler's [fdtd]
# group — the "[...]" would be glob-expanded and a ">=" pin redirected. `-U`
# fetches recent, Python-3.12-compatible builds. Keep in sync with the [fdtd]
# group in pyproject.toml.
matplotlib
numba
#
# phidler itself is appended by packaging/appimage/build.sh as the absolute path
# to the current checkout — so the AppImage bundles this exact source. Bundling
# the FDTD stack at build time is deliberate: an AppImage is a read-only mount,
# so the app's "install photonfdtd on demand" pip path can't run inside it.
